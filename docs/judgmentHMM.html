judgmentHMM Module¶

A small framework to create and validate different models of judgments. Provides easy setup of parameter sweeps, model performance metrics and model comparison. Data are passed as CSV.

class judgmentHMM.Experiment(resultDir='results', filename=False)[source]¶

Set up and run one or a series of experiments (creating a model with certain parameters, validating the model, calculating the performance metrics). Results are saved as csv, where each line corresponds to one experiment / parameters setting. If no filename is given for the resulting csv file, the current time will be used.

run(args, models, permute=False, maxParams=3)[source]¶

Run the experiment(s).

Args:

args (dict): Dictionary key/value definition of experiment arguments / parameters. If values are given as lists instead of single strings or numbers, all permutations of all given parameters constellations will be run as single experiments. Defined keys are:

ratedTurns (str) -- location of labeled input data in csv format
validationMethod (str) -- method to use for validation, eg.: "lolo" (leave-one-label-out cross-validation), methods are defined in RatingModels._getCV()
quantizationLevel (str) -- Defines how features are clustered. Two approaches are available: a) Cluster each feature individually (nxk clusters), b) quantize all features as one n-dim feature (k clusters). Codes are a) "parameters", b) "features".
k (int) -- Number of clusters to use for quantization
modelParams (list) -- List of features (aka parameters) to use in model. Have to correspond to column headers in input csv.
cleanUp -- Remove unlabeled turns (turns without any judgment)
removePauses -- Remove turns where neither system nor user is uttering anything
missing -- Replace missing values with given value, e.g. np.nan

permute (bool): Whether to use permutation as defined above.

models (dict): Definition of models to investigate. Key: Name of model, value: dict of special model parameters. Models are defined in class RatingModel, functions _test and _train.

maxParams (int): Maximum number of features to use in model.

class judgmentHMM.TestResults(name, indicators=False)[source]¶

Class in which performance metrics are defined and which can be used to calculate performance of true vs predicted series of judgments. Example usage:

results = TestResults(model.name) for test,true in data:

predicted = model.predict(test) results.compare(true,predicted)

metrics = results.getResults() print(‘MAE of ‘ + model.name + ‘:’) print(metrics[‘MAE’])
